> OS provides a interface btw User/Software and Hardware
### Main functions of OS:
1. Resource Management
2. Process Management (CPU Scheduling)
3.  Storage Management (HDD), file system
4. Memory Management (RAM)
5. Security (data of one app doesn't collides with another or leaks)

### Types of OS:
1. Batch
		back in 1960s things were pretty manual you needed to paper tape and and write your jobs in it, that computer does and this was done  by punching the paper and this paper with all jobs of yours was then called a batch and it was given to computer along with batches of others too
		The Major con was that until a job is completed CPU remained idle
		Computers: Fortran, IBSYS709x
2. Multi-programming
		Processes are **non-preemptive**,
		meaning when there are multiple processes coming CPU will move to next process only when a process is completed, and it will move to next process iff process tells itself to other tasks like I/O then the next process is taken
		Reduces **idleness** of CPU
3. Multi-tasking/ Time-sharing
		Processes are preemptive,
		Every process is given a particular amount of time to complete, and if the process is not completed then it is re-scheduled for later execution, idleness is less here similar to multi-programming but the main benefit is **responsiveness**
		Moderns computers are based on this only
4. Real time OS
		In this OS time of output is the main concern, process should be completed in real time
		Time matters most, delays are not expected
		Types-
			**Hard**: slightest of time delay is not tolerable it can create a **critical situation** if time is delayed, e.g. Missile system, Flight simulation
			**Soft**: time delay is not expected here also but in this critical situation will not arise due to delay, e.g. gaming, live-streams
5. Distributed Environment
		Machines are present at loosely coupled environment
		They are at different geographical locations, working on same thing
		If one fails its work can be done by other
		e.g. Backups plans are usually stored in distributed environment
6. Clustered
		Multiple machines are connected through a local network, acting as a server
		e.g. super-computer, computer labs, cloud computers
7. Embedded
		Machines that work on fixed functionality, cannot do other processes
		e.g. microwave, fridge, washing machine

### System Call

**File related**
	open(), read(), write(), close(), create file, etc.
**Device related**
	read, write, reposition, ioctl, fcntl, etc.
Information
	getPid, attributes, get System time and data, etc.
Process Control
	load execute, abort, **Fork**, wait, signal, allocate, etc.
Communication
	Pipe(), create/delete connections, Shmget(), etc.
